# Episode cover formula

Treat episode cover art as a required podcast deliverable when `episode_cover.enabled` is true.

## Formula

Build every cover from these layers, back to front:

1. Approved brand template or visual system.
2. Episode-specific hero artwork that accurately represents the episode.
3. Contrast treatment that preserves the focal subject while supporting readable text.
4. Show, series, or story identity.
5. Episode title as the primary text hierarchy.
6. Episode number and content label when applicable.
7. Creator or brand lockup.

Do not substitute generic show art, unrelated prior artwork, or a video thumbnail unless the project configuration explicitly allows it.

## Production sequence

1. Lock the approved episode title, series, episode number, content label, and brand spelling.
2. Confirm source-art and font licenses.
3. Create one high-resolution master using the configured canvas, safe margin, template, and hierarchy.
4. Derive platform variants from the master without stretching, cropping essential text, or changing approved visible metadata.
5. Export in the configured format and color mode.
6. Render the configured small-preview size and inspect readability.
7. Record the master and derivative file identities.
8. Show the exact cover to the user and stop for visual approval.

## Approval boundary

Cover approval applies only to the exact exported file and visible metadata shown to the user. Re-run QA and request approval again after changing artwork, crop, title, episode number, label, logo, font, color, or layout.

Video production requires both current audio approval and current cover approval. Neither approval substitutes for the other.
